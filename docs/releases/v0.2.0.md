# ShareClaw v0.2.0

ShareClaw v0.2.0 turns the project from a promising shared-memory utility into a much more complete runtime for self-improving agent systems.

## Highlights

- lock-safe shared-state writes for local multi-agent coordination
- markdown mirrors for brain, tasks, decisions, and events
- first-class task queue and consensus APIs
- richer CLI with task, consensus, event, and files commands
- runnable launch-swarm demo
- synthetic launch benchmark
- CI plus expanded test coverage

## What Changed

### Core runtime

- `Brain` now supports task creation, pickup, completion, requeueing, consensus decisions, voting, and resolution
- shared state is protected with file locking and fresh reloads before writes
- markdown mirrors are generated automatically for agents and humans
- reports and prompt context now include tasks, decisions, and recent events

### Developer experience

- CLI now exposes the coordination primitives directly
- templates expanded beyond the shared brain to include tasks, events, and decisions
- integrations docs updated to reflect the generated markdown mirrors
- GitHub Actions now runs tests on multiple Python versions

### Launch assets

- `examples/launch-swarm/run_demo.py` gives a narrative end-to-end demo
- `benchmarks/launch_swarm.py` provides measurable proof of the compounding-learning thesis
- public launch guide and reusable launch copy are now included in the repo

## Verification

## Upgrade Notes

This release is still intentionally small and zero-dependency at runtime. The system remains file-based, but now behaves much more like an agent operating system:

- shared brain
- task queue
- consensus
- events
- handoffs
- disciplined experiment loop

## Suggested Next Step

Use the launch-swarm demo as the first public proof of the system, then adapt the same pattern for coding swarms, content swarms, and research swarms.
